# ポーズ品質ワークフロー

この文書は、AIが作る3D人型ポーズで腰浮き、逆関節、奥行きの取り違え、身体の貫通を繰り返さないための標準手順です。

## 1. 作成前に意味と支持条件を定義する

ポーズ名だけでなく、床・椅子・物体へ接触する身体部位と、どこで体重を支えるかを先に決めます。

例: 床座り

- 骨盤下面が床へ接触
- 両足または膝の指定点が床付近
- 上半身の重心が支持範囲から大きく外れない

## 2. 骨長固定で関節を解く

手足はニュートラルリグの骨長を維持した2ボーンIKまたは骨長固定計算で作ります。関節座標を自由配置したまま完成扱いにはしません。

## 3. 接触点をJSONへ保存する

`contact_points` に支持点を明示します。ビューアは緑のマーカーとして表示します。

## 4. 幾何検査

最低限、次を確認します。

- 最大骨長誤差
- 床より下へ入った関節数
- 膝と肘の曲がる方向
- 左右脚の前後関係
- 明らかな自己貫通
- 座りポーズの骨盤下面と支持面の関係

## 5. 四視点検査

同じポーズを次の視点で確認します。

- 正面
- 側面
- 上面
- 斜め

正面だけで合格にはしません。側面では接地、上面では脚・腕の前後関係、斜めでは全体の読みやすさを確認します。

## 6. ブラインド意味評価

利用可能な場合、作成目標とファイル名を隠し、別コンテキストの評価役へ四視点画像だけを渡します。

評価質問:

1. この人物は何をしているか
2. どの身体部位で体重を支えているように見えるか
3. 不自然な接触、逆関節、ねじれ、貫通がないか
4. 確信度

目標と認識が一致しない、または重大な問題が指摘された場合は修正して再評価します。

## 7. 公開条件

- 骨長検査を通過
- 床下への重大なめり込みなし
- 必須接触が成立
- 四視点確認済み
- ポーズの意味が読み取れる

独立したブラインド評価を実行できない環境では、JSONの `independent_blind_agent_review` を `pending` のままにして過大評価しません。

## 8. 公開確認URL

各ポーズは次の形式で直接開けます。

```text
https://nemotosiki.github.io/kimodo-pose-viewer/?pose=<pose_id>
```

ポーズを追加・修正した応答では、最後にクリック可能な確認URLを必ず掲載します。
