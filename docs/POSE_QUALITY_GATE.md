# ポーズ品質ゲート

この文書を、今後このリポジトリでポーズを作成・修正するときの標準手順とする。

## 目的

ポーズ作成者の「意図どおりに見えるはず」という思い込みを排除し、次の2点を別々に検査する。

1. **意味が伝わるか** — 匿名画像だけを見た評価役が、目標の動作・姿勢を正しく説明できるか。
2. **身体として成立するか** — 接地、骨長、関節方向、支持状態、めり込みに重大な破綻がないか。

数値検査だけで「喜んでいる」「あぐら」などの意味は判定できない。一方、視覚評価だけでは骨の伸縮や数cmの浮きを見逃すため、両方を必須とする。

## 役割

- **Author**: 目標を知り、ポーズを作る。
- **Mechanical Validator**: 目標を参照せず、骨長・接地・床貫通などを検査する。
- **Blind Semantic Reviewer**: 目標名、説明、ファイル名を見ず、4視点画像だけからポーズを説明する。
- **Naturalness Reviewer**: 目標を知らず、人体として不自然な点を列挙する。
- **Judge**: 目標と各評価結果を比較して合否を決める。

同じ会話・同じコンテキストでAuthorとReviewerを兼任した結果は、参考にはできるが**独立レビューとしては扱わない**。独立したモデル、別コンテキスト、または人間のレビューが完了するまでは `provisional` とする。

## 必須ワークフロー

### 1. 目標と接触条件を先に定義

`authoring-target.json` に次を記録する。

- 姿勢・動作の目標
- 必須支持点（尻、足裏、手など）
- 近接すべき部位
- 左右、前後、視線、感情
- 許容する非対称性

座りポーズでは `Hips` 関節ではなく、骨盤プロキシの下端を床・座面との接触点として扱う。

### 2. 骨長を固定して解く

関節座標を自由配置したまま完成としない。

- ニュートラルリグの各親子距離を保存する。
- 腕と脚は2ボーンIKまたは同等の幾何計算で解く。
- 肘・膝の曲がる側はポール方向で指定する。
- 直接座標を使う場合も、最終的な骨長誤差を検査する。

### 3. 機械検査

最低限、次を `validation.json` に出力する。

- 最大骨長相対誤差
- 必須接触点と支持面の距離
- 床より下へ入った関節・身体プロキシ
- 肘・膝の角度
- 左右の膝・肘の曲がる方向
- 支持点の凸包に対する重心投影の概算
- 明白な自己貫通

### 4. 匿名4視点レンダリング

同じポーズを以下の4方向から生成する。

- 正面（直交投影）
- 側面（直交投影）
- 上面（直交投影）
- 斜め（全体評価）

画像には目標名、説明、意味を示すUIを含めない。匿名IDだけを使用する。関節線だけでなく、骨盤・胸郭・頭の身体プロキシ、床面、接触点を表示する。

### 5. ブラインド意味評価

Reviewerには匿名4視点画像だけを渡し、次を質問する。

1. 人物は何をしているか。
2. `standing / chair_sitting / floor_sitting / squatting / kneeling / lying` のどれか。
3. どの部位で身体を支えているように見えるか。
4. 第1候補、第2候補、確信度0〜100。

### 6. 自然さ評価

別のReviewerに、目標を知らせず次を評価させる。

- 腰や尻が浮いていないか
- 肘・膝が逆方向や極端な方向へ曲がっていないか
- 足、手、骨盤が不自然にねじれていないか
- 身体同士や床を貫通していないか
- 重心と支持点が矛盾していないか
- 4視点の間で解釈が食い違わないか

### 7. 合否判定

通常公開へ進める最低条件:

- 目標の大分類が第1候補と一致
- 具体的な動作が第1候補または第2候補に含まれる
- 意味評価の確信度が75以上
- 重大な自然さ指摘が0件
- 必須接触点が許容差内
- 最大骨長相対誤差が0.5%以下
- 逆関節、重大な床貫通、重大な自己貫通が0件

同じコンテキストによる自己評価しかない場合は、上記を満たしても `provisional` のままにする。

### 8. 修正ループ

不合格時は、評価文を修正入力へ変換する。匿名IDを更新し、新しいReviewerへ再提出する。最大3周で通らなければ人間確認へ回す。

## 状態

- `draft`
- `mechanically_validated`
- `rendered`
- `provisional`
- `review_failed`
- `review_passed`
- `published`

通常のポーズ一覧へ入れてよいのは `review_passed` だけ。`provisional` はレビュー専用URLでのみ公開できる。

## 推奨ファイル構成

```text
site/reviews/<anonymous-pose-id>/
├─ authoring-target.json
├─ pose.json
├─ validation.json
├─ blind-review-request.json
├─ semantic-review.json
├─ naturalness-review.json
├─ review-status.json
├─ four-view.svg
└─ index.html
```

## ブラインド評価用固定プロンプト

```text
添付画像は、同じ3D人型を正面・側面・上面・斜めから見たものです。
画像内のファイル名や説明を手掛かりにせず、身体の形だけから答えてください。

1. この人物は何をしていますか。
2. posture_classを standing / chair_sitting / floor_sitting / squatting / kneeling / lying から選んでください。
3. どの身体部位で体重を支えているように見えますか。
4. 第1候補、第2候補、確信度0〜100を答えてください。
5. 腰の浮き、逆関節、ねじれ、貫通、重心の矛盾があれば列挙してください。
```
